# CPE Benchmark Configs

This directory contains CPE configuration and system prompt artifacts used by the Harbor and Pier CPE agents.

The adapters are intentionally generic: they install CPE, install a config artifact to `$HOME/.config/cpe/cpe.yaml`, install a system prompt artifact to `$HOME/.config/cpe/agent_instructions.md`, prewarm the Go module cache used by CPE's `execute_go_code` tool, and then run CPE with explicit `--config` and `--model` values. Artifacts may be HTTP(S) URLs or local `file://` URLs; local files are embedded into the setup command so mandatory smoke tests validate the current worktree. Add new experiment variants here by creating a new subdirectory with a `cpe.yaml` and prompt files. Always pass the CPE profile ref explicitly with `--ak model_ref=<ref>`; pass `--ak thinking_level=<value>` only when that profile declares `thinkingValues`. The benchmark framework's `-m` model name is not used for inference. Pin the CPE release with `--ak version=<tag>` for reproducible runs.

The bundled GLM configs target Z.ai's OpenAI-compatible endpoint using CPE's implemented `openai` provider type and expose `high` as their smoke-test thinking value. The `text_edit` prompt tracks the live CPE agent instructions from `$HOME/Library/Application Support/cpe/agent_instructions.md`; the `execute_go_code_edits` variant removes references to unavailable text-editing tools while preserving the execute-Go-code safety rules.

The `deep-swe` config contains multiple model profiles. Every profile uses the same runtime prompt path, `$HOME/.config/cpe/agent_instructions.md`; choose the actual prompt by passing the desired local or HTTP artifact as `system_prompt_url`. Use `gpt_instructions.md` for GPT profiles and `agent_instructions.md` for non-GPT profiles. GPT profiles use CPE's OpenAI Responses OAuth path for ChatGPT/Codex-compatible backends: `type: responses` with `auth_method: oauth` and explicit `base_url` values. Pass the host CPE auth store as an `auth_url` agent kwarg for GPT profiles so Harbor/Pier can install it into the Linux sandbox runtime config directory. On this macOS machine, the host auth file URL is `file:///Users/shashankpachava/Library/Application%20Support/cpe/auth.json`. The Pier adapter includes selected profile runtime hosts and OAuth token hosts in the network allowlist.

Use local `file://` URLs while iterating on active experiments. Use commit-SHA raw GitHub URLs for final reproducible benchmark runs. For Pier, the network allowlist includes install/cache domains and runtime domains derived from the selected `model_ref` profile; set explicit `base_url` values for non-default provider endpoints.
