# CPE Benchmark Configs

This directory contains CPE configuration and system prompt artifacts used by the Harbor and Pier CPE agents.

The adapters are intentionally generic: they install CPE, install a config artifact to `$HOME/.config/cpe/cpe.yaml`, install a system prompt artifact to `$HOME/.config/cpe/agent_instructions.md`, and then run CPE with explicit `--config`, `--model`, and `--thinking-level` values. Artifacts may be HTTP(S) URLs or local `file://` URLs; local files are embedded into the setup command so mandatory smoke tests validate the current worktree. Add new experiment variants here by creating a new subdirectory with a `cpe.yaml` and `agent_instructions.md` pair. The bundled prompts track the live CPE agent instructions from `$HOME/Library/Application Support/cpe/agent_instructions.md`; the execute-Go-code variant removes references to unavailable text-editing tools while preserving the execute-Go-code safety rules. Always pass the CPE profile ref and thinking value explicitly with `--ak model_ref=<ref>` and `--ak thinking_level=<value>`; the benchmark framework's `-m` model name is not used for inference. Pin the CPE release with `--ak version=<tag>` for reproducible runs. The bundled GLM configs target Z.ai's OpenAI-compatible endpoint using CPE's implemented `openai` provider type and expose `high` as their smoke-test thinking value.

Use local `file://` URLs while iterating on active experiments. Use commit-SHA raw GitHub URLs for final reproducible benchmark runs.
